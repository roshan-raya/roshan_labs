Vulnerability explanation

Enter credentials or use an example payload. In Vulnerable mode, injection-like input can change the query logic.

Preventing SQL injection

  • Never concatenate user input into SQL strings.
  • Use parameterized queries / prepared statements so the database treats input as data, not code.
  • Validate and sanitize input; use least-privilege database accounts.
  • Consider a WAF and security logging for detection.

Python — parameterized query

import sqlite3

conn = sqlite3.connect('app.db')
cursor = conn.cursor()

# Safe: placeholders, then pass values separately
username = request.form.get('username')
password = request.form.get('password')
cursor.execute(
    "SELECT * FROM users WHERE username = ? AND password = ?",
    (username, password)
)
